Nova tarefa Issues Repositório

Tarefas pessoais e da empresa, num só lugar.

Sistema leve baseado em GitHub Issues e Projects: formulários prontos, labels parametrizáveis e automação quando o título indica o tipo de demanda.

Fluxo de uso

  1. Nova demanda Abre uma Issue (idealmente pelo formulário).
  2. Labels e prioridade Ajusta labels; o workflow pode aplicar a principal pelo título.
  3. Project Coloca o cartão no quadro (ex.: Inbox → Em andamento).
  4. Colunas sugeridas Inbox, Planejado, Em andamento, Aguardando, Concluído (editáveis no JSON).
  5. Fechar Ao concluir, fecha a Issue e atualiza o board.

Desenvolver e manter

A aparência desta página fica em docs/. O comportamento do sistema (labels, mapeamento título → label, colunas sugeridas) está em config/parametros.json. No clone, rode validação e scripts conforme o README do repositório. 

cd COD---CONTROLE-OBRIGA-ES-DIARTIA
.\scripts\validar-parametros.ps1
.\scripts\verificar-formularios-issues.ps1
.\scripts\setup-labels.ps1   # requer GitHub CLI (gh)
